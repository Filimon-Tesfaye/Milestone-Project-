A Short History of HTTP

The HyperText Transfer Protocol (HTTP) was originally created in the early 1990s at CERN as part of Tim Berners‑Lee’s work on the World Wide Web. The goal was simple: define a lightweight protocol that allowed a client (a web browser) to request hypertext documents from a server.

Early Web and HTTP/0.9

The first documented version of HTTP is sometimes called HTTP/0.9. It was extremely simple and supported only one method: GET. Requests did not include headers, and servers responded with just the raw HTML body. There was no concept of status codes or content negotiation.

Early HTTP Milestones
Year (approx.) Milestone Notes
1990–1991 Prototype HTTP and first web server Used at CERN to share research documents.
1993 HTTP/0.9 widely used Simple text‑based protocol, only GET.
1996 HTTP/1.0 described in RFC 1945 Introduced headers, status codes, and more methods.
1997 HTTP/1.1 published in RFC 2068 (later 2616) Became the dominant version of HTTP for many years.

Standardization Through RFCs

Over time, HTTP was standardized through a series of Request for Comments (RFCs). HTTP/1.0 was documented in RFC 1945, and HTTP/1.1 was later refined and clarified in a set of specifications that replaced the original single RFC. These documents defined the request–response model that browsers and servers still use today.

From Simple Documents to a Full Web Platform

As the web grew from static documents to complex applications, HTTP had to support new use cases: image loading, forms, cookies, caching, streaming, and APIs. This growth eventually motivated new protocol versions (HTTP/2 and HTTP/3) that focused on performance and reliability on modern networks. Those versions are discussed in more detail on the HTTP Evolution page.
